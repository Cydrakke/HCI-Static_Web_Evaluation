Task Overview

Description of the evaluation methodology and task selection

Outlines the tasks that were used during our evaluation of the Mobile Store Management System. These tasks were selected to represent typical user interactions with the system and to cover the main functionality areas. Each task was performed by 5 evaluators, and the results were documented in terms of task success, completion time, and observed issues.

Evaluation Conclusion

Summary of Findings

Based on the task performance data and observed user behavior, we can draw the following conclusions:

  • The system is mostly functional and can complete core task, such as creating accounts, managing products, and placing orders without critical failures. Lack of feedback for some interactions that can lead to user frustration
  • Security vulnerabilities and inconsistent input Sanitation pose as a threat, but sql injection were blocked on a login page.
  • The overall UI and the display content need an improvement and optimize sizes of content like banner. There is a performance issue during product sorting and searches that may affect user satisfaction.
  • Inability of the system to track placed orders and the system allows checkout with an empty cart.

Overall, the evaluation revealed that the system is mostly functional and enables user to complete key tasks, there are significant usability and security concerns. The system's major strengths lie in its core task functionality and stable payment integration, but weaknesses include poor input Sanitation, lack of user feedback mechanisms, and missing features like order tracking and undo capabilities.Solving These issues will surely enhance both security and user experience of the system.
