Proposed Improvements

Recommended UI improvements based on evaluation findings

Improvement Strategy

Overview of our approach to addressing the identified usability issues

Based on the usability issues identified during our evaluation, we have developed a set of proposed improvements for Mobile Store Management System. These improvements are designed to address the most critical usability issues while maintaining the overall look and feel of the website.

UI Improvements

Consistent color theme

Related Heuristic: Consistency and Standards

Description:

A consistent color theme for the side menu in a mobile store management system ensures a unified and cohesive design.

Justification:

Consistent colors improve navigation, reduce confusion, and create a professional look, making the app easier to use and enhancing brand recognition.

Proper Error messages and validation

Related Heuristic: Error Prevention

Description:

It lacks a message info that prevents users from modifying the data when it is in admin mode this can lead to unwanted deletion of data leading to error.

Justification:

Without a clear warning in admin mode, users may unknowingly modify or delete important data. This violates the error prevention heuristic, as the system fails to alert users before risky actions, increasing the chance of accidental errors.

Visibility of the system status

Related Heuristic: Visibility of System Status

Description:

The message status is too small and the color for the messages needs to use the standard like red for error and green for success.

Justification:

Small and non-standard message indicators reduce the visibility of system feedback. Using clear sizing and conventional colors (like red for errors, green for success) helps users quickly understand system status and respond appropriately.

Consistency and Standards

Related Heuristic: Consistency and Standards

Description:

The banner takes up excessive screen space, pushing important content out of user view, it needs to optimize the banner size for better user experience.

Justification:

The banner takes up too much screen space, pushing important content out of view. Reducing its size would improve user experience by keeping essential content accessible and preventing unnecessary scrolling.

Visibility and System Status

Related Heuristic: Visibility and System Status

Description:

Navigation menu on the header tab is misaligned, it needs to adjust their layout to be aligned.

Justification:

The navigation menu on the header tab is misaligned, which affects the overall layout. It should be adjusted to ensure proper alignment for a cleaner, more organized appearance.

• There is no "Undo" button right after clicking the "Empty Cart" or "+" and "-" Affects user control and can lead to frustration or loss of data.
• Does not provide confirmation before saving the entry May lead to unintentional changes or errors being saved.

• The banner occupied half of the website's display Affects user experience but not core functionality; should be resized for better content visibility.
• Color theme between side menu (purple) and header (pink) is inconsistent Aesthetic inconsistency that affects visual coherence and professionalism.
• Lack of visual cues—just a plain text interface Makes the UI less intuitive but does not directly break functionality.
• No On-Screen Help or Hints Reduces usability for new users but doesn’t affect experienced users as heavily.
• Lack of currency symbols Can cause confusion but doesn't impact transaction mechanics directly.
